Porting Source 1 map to Source 2 (For CSGO to CS2)

Converting map & it's content to the editor
  • Set-up folders (csgo filepath w/ precompiled content & gameinfo file; path to the source files (.vmf & assets)(folder path cannot contain spaces); CS2 gameinfo file)
  • Install Python
  • Set-up PATH environment variable
  • Install colorama (pip install colorama)
  • Create new addon with the desired name with the workshop tools
  • Locate import script
  • Modify import script
  • Run import script GUI
Outliner (prefabs) & texture fixing

The script will give out 3 different .vmap (map.vmap; map_prefab.vmap; map_environment_prefab.vmap (This last one will only be there if you use -usebsp))
You can combine them all by collapsing the prefabs.
Replace any surface specific clip textures with the simple clip material
If you have any custom sprites they have to be manually imported; create a .vtex file in the content folder:

Lighting & light probes

Check if you have any light_omni2 with crazy high brightness & range values and manually fix them. You can use indirect light scale to make a light similar to constant light in source.
The S2 importer will approximate the light probes & cube maps combined entities; you're better off remaking them manually; or if you don't add them, then the lighting will look very bad on non-lightmapped objects. Be sure to make light probe entities touch each other, and to increase resolution size if some lighting looks bad. (If you want bad/retro-looking lighting, uncheck stationary light shadows on the light_environment

Navigation mesh

Just go to Navigation ->place a Create Walkable Seed anywhere in the playing area ->Nav Preview
You can tweak the navigation mesh by placing nav-clip brushes

Sounds

Sounds will not be ported with the script. You will need to import them to a folder of your choosing, then edit the soundevents_addon.vsndevts file by adding the sound file with the desired settings. Add a point_soundevent with the desired soundscape sound and check play to local player and start on spawn.
Or just modify the soundscape, and replace all proxys to be normal soundscapes for them to work.

Radar/Mini-map

Add a cs_minimap_boundary at the lowest-south-west corner of the map and the highest-north-east corner of it. You can add multiple to generate multiple layers. You can change the resolution of the generated minimap with cs_minimap_output_size "square-resolution"
Create a new material using the material editor with the same name as the radar file, as a composite generic, then add the generated image(s) to the material.
Then edit the map .vtex file name:
mapname_radar to mapname_radar_tga
copy the same file and rename it to mapname_radar_spectate_tga

If you have a multi layered minimap, you need to edit the mapname.txt in the game addons/resource/overview folder

"mapname"
{
	"autogenerated_tga"		"1"
	"pos_x"		"-1504"
	"pos_y"		"3560"
	"scale"		"4.343750"
	
	"verticalsections"
	{
		"default" // use the primary radar image - highest point
		{
			"AltitudeMax" "10000"
			"AltitudeMin" "-80"
		}
		"1" // mapname_1_radar - middlest point
		{
			"AltitudeMax" "-80"
			"AltitudeMin" "-416"
		}
		"0" // mapname_0_radar - lowest point
		{
			"AltitudeMax" "-416"
			"AltitudeMin" "-10000"
		}
	}
}

Finishing Touches
