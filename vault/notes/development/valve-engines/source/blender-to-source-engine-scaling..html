Blender to Source Engine scaling.

https://developer.valvesoftware.com/wiki/Dimensions_(Half-Life_2_and_Counter-Strike:_Source)

Maps, architecture, and some prop models use a scale of 1 foot = 16 units.
  1. We start with the given information:
    • 1 unit in Blender = 1 meter
    • In the Source engine (for this scale), 16 units = 1 foot
  2. We need to convert meters to feet:
    • 1 meter = 3.28084 feet (this is a standard conversion)
  3. Now we calculate how many Source units are in 1 meter:
    • If 16 Source units = 1 foot, and 1 meter = 3.28084 feet
    • Then 1 meter = 16 * 3.28084 Source units
    • 16 * 3.28084 = 52.4934
  4. This 52.4934 is our scaling factor. It tells us how many Source units are equivalent to 1 Blender unit (1 meter).
Human character models and certain other models for Source currently use 1 foot = 12 units.
  • We start with the given information:
    • 1 unit in Blender = 1 meter
    • In the Source engine (for this scale), 12 units = 1 foot This is the key difference from the previous calculation, where it was 16 units = 1 foot.
  • We need to convert meters to feet:
    • 1 meter = 3.28084 feet (this is a standard conversion, same as before)
  • Now we calculate how many Source units are in 1 meter:
    • If 12 Source units = 1 foot, and 1 meter = 3.28084 feet
    • Then 1 meter = 12 * 3.28084 Source units
    • 12 * 3.28084 = 39.37008 (which rounds to 39.3701)
  • This 39.3701 is our scaling factor. It tells us how many Source units are equivalent to 1 Blender unit (1 meter) for this scale.
